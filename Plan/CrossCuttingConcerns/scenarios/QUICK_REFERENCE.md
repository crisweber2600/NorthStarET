## Specification Quality Checklist

Before approving a specification, verify:

- [ ] **Business Value**: Clear statement of value proposition
- [ ] **Scenarios**: Given/When/Then scenarios cover key use cases
- [ ] **User Stories**: Prioritized with P1/P2/P3 labels
- [ ] **Functional Requirements**: Specific, measurable requirements
- [ ] **Success Criteria**: Clear metrics for completion
- [ ] **Technical Plan**: Architecture, technology stack, phases identified
- [ ] **Layer Assignment**: Correctly identified (CrossCuttingConcerns)
- [ ] **Dependencies**: Shared infrastructure usage documented
- [ ] **Performance Targets**: Latency, throughput, resource usage defined
- [ ] **Test Strategy**: Unit, integration, E2E test approach specified
- [ ] **Constitutional Compliance**: All 7 principles addressed

## Implementation Workflow (After Specification Approval)

### Phase 0: Preparation
1. Review and approve specification on proposed branch
2. Resolve any branch issues (see RESOLUTION_ACTION_PLAN.md)
3. Checkout spec branch: `git checkout CrossCuttingConcerns/[###]-feature-name-spec`

### Phase 1: Task Generation
1. Run `/speckit.tasks` to generate `tasks.md`
2. Review task breakdown with team
3. Commit and push tasks.md

### Phase 2: Implementation Setup
1. Run `/speckit.implement` to create implementation branch
2. Switches to: `CrossCuttingConcerns/[###]-feature-name`
3. Begin Phase 1 development

### Phase 3: TDD Development
For each phase in tasks.md:
1. **Red**: Write failing tests (capture output)
2. **Green**: Implement minimal code to pass (capture output)
3. **Refactor**: Improve code quality
4. **Review**: Phase checkpoint before continuing

### Phase 4: Quality Gates
At each phase boundary:
- [ ] ‚â•80% test coverage
- [ ] All tests passing (unit, integration, E2E)
- [ ] Red‚ÜíGreen evidence captured
- [ ] Push to phase review branch: `git push origin HEAD:[###]-feature-name-review-Phase[N]`

### Phase 5: Completion
After all phases:
- [ ] Final code review
- [ ] Security scan (CodeQL)
- [ ] Performance verification
- [ ] Documentation updated
- [ ] Merge to main
